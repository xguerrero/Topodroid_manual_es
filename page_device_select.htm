SELECCIÓN DE DISPOSITIVO

TopoDroid es compatible con el siguiente dispositivo:
  • DistoX (primera versión)
  • DistoX2 (segunda versión)
  • Sap5 (Shetland Attack Pony), solo modelos con bluetooth
  • Bric4 (cartógrafo integrado reforzado con Bluetooth)
Todos los dispositivos deben estar emparejados con Android.

Normalmente, TopoDroid funciona con un solo dispositivo. Los datos se pueden descargar de forma continua (modo continuo) o por lotes (modo bajo demanda). La diferencia entre los dos modos es que el modo continuo permanece activo cuando la descarga de datos finaliza, o se pierde la conexión con el dispositivo, mientras que el modo bajo demanda está cerrado.

Sap5 y Bric4 descargan los datos solo en modo continuo.

Con TopoDroid de DistoX se puede trabajar con más de un dispositivo a la vez, aunque los datos todavía se descargan desde un solo dispositivo a la vez.
El modo de descarga de datos debe establecerse en multi. Cuando es así, un toque prolongado en el botón descargar abre un cuadro de diálogo para elegir de qué DistoX descargar los datos.

Shetland Attack Pony
TopoDroid soporta el modelo de Shetland Attack Pony, versión 5 (Sap5), con bluetooth.
El dispositivo debe haber sido previamente "emparejado" con Android, utilizando la aplicación Configuración. El PIN es "000000" (seis ceros).
Posteriormente, el Sap5 aparece entre los dispositivos conocidos en la Ventana de dispositivos. Aparece como "Dirección MAC de SAP XX", donde "XX" es el código de dos letras de Sap5, por ejemplo, "HG".
Para seleccionar el Sap5 como dispositivo actual, toque su entrada en la lista.

Mientras está conectado, el Sap5 transferirá la lectura de los valores de un disparo a la aplicación, ya sea que los datos del disparo sean o no se almacenan o desechan en el dispositivo.

El protocolo de transmisión de datos Sap5 no incluye acuse de recibo de los datos. Por lo tanto

  • si Sap5 no está conectado, los valores de disparo no se transfieren a la aplicación.
  • al volver a conectarse, el Sap5 no transfiere los datos de toma almacenados mientras no estaba conectado.

El Sap guarda en su memoria interna los datos que han sido "almacenados". Estos se pueden recuperar como un archivo Survex, usando el programa Sap PonyTrainer.

La calibración del Sap5 la realiza el dispositivo y no requiere un programa externo.

Bric4
TopoDroid es compatible con la versión Bric4.
El dispositivo debe haber sido previamente "emparejado" con Android, utilizando la aplicación Configuración.
Posteriormente, el Bric4 aparece entre los dispositivos conocidos en la Ventana de dispositivos. Aparece como "Dirección MAC del código Bric4", donde el código es el código de cuatro dígitos del Bric4, por ejemplo "0039".
Para seleccionar el Bric4 como dispositivo actual, toque su entrada en la lista.

Una vez conectado a un cliente, el Bric4 recuerda la conexión, incluso si está apagado, por lo que permanece "siempre conectado".
Cuando TopoDroid está conectado, las lecturas de toma se transfieren inmediatamente a la aplicación. Si TopoDroid no está conectado, los datos se guardan en la memoria y se transfieren a la aplicación cuando está próximo conectado.

El protocolo de transmisión de datos Bric4 incluye reconocimiento por parte de Android,pero no incluye reconocimiento de la recepción de los datos por parte de la aplicación. El protocolo tiene unos segundos de retraso entre la conexión y la primera transferencia de datos, y TopoDroid tiene un segundo de retraso después del último dato, para cerrar la conexión. Por este medio normalmente no se pierde ningún toma, aunque no hay garantía.

El Bric4 almacena los datos en la memoria interna en archivos CSV. Si está conectado a una PC a través de USB, el dispositivo aparece como memoria externa y los archivos CSV se pueden copiar a la PC.

La calibración del Bric4 la realiza el dispositivo y no requiere un programa externo.

El Bric4 se puede controlar desde la aplicación. En particular, es posible encender/apagar el láser y tomar tomas El Bric4 también tiene un "modo de exploración" en el que se toman lecturas de forma continua, aproximadamente dos por segundo.

TopoDroid puede seguir tres protocolos de datos diferentes (según una configuración en el conjunto "personalizado"). El primero recibe solo los datos primarios (distancia, azimut y clino) que no incluyen el número de ID de datos asignado por el Bric4. Los otros dos reciben los datos completos. Difieren si TopoDroid utiliza el número de identificación proporcionado o asigna sus propios números.
Cuando se reciben los datos completos, TopoDroid registra los códigos de error y los valores de Bric4 en el comentario de disparo.
