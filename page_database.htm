BASES DE DATOS TopoDroid

Es útil tener una idea aproximada sobre la organización de la información y los datos en las bases de datos. Las unidades de trabajo en TopoDroid son los levantamientos y las calibraciones. Un levantamiento es una sola sesión de levantamiento de un trozo de cueva. Una calibración es una única calibración DistoX.
Las topografías se almacenan en la base de datos "distox14.sqlite". Hay una base de datos de este tipo en cada carpeta de proyecto.
Las calibraciones se almacenan en la base de datos "device10.sqlite", junto con la información del dispositivo. Esta base de datos es global y se almacena en el TopoDroid carpeta privada de "archivos".

La tabla más importante en la base de datos de la topografía es:

  • topografías: información de topografías

Los registros de topografías almacenan el nombre de la topografía, la fecha, el equipo, la declinación magnética, el nombre de la estación inicial y posiblemente un comentario.
Relacionadas con las topografías están las tablas para:
  • Tomas: datos de la topografía
  • estaciones: estaciones guardadas
  • gráficos: croquis 2D; planos, perfiles, cortes en x
  • fotos; imágenes asociadas a tomas de topografías
  • sonidos; comentarios de audio asociados a tomas de topografías
  • sensores: medidas asociadas a tomas de levantamiento
  • fijas: (estaciones geolocalizadas)

Las tablas más importantes en la base de datos del dispositivo son

  • dispositivos: información de los dispositivos
  • calibs: información de calibraciones

Los registros de calibración contienen el nombre de la calibración, la fecha, el dispositivo DistoX, el algoritmo y los resultados del último cálculo: coeficientes, error promedio, error máximo, número de iteraciones.

Relacionada con las calibraciones está la tabla de datos

  • gms: datos de calibración

Finalmente cada base de datos tiene la tabla

  • configs: valores de configuración

Puede obtener más información sobre la organización de los datos en la base de datos mirándolos con un visor de SQLite.
