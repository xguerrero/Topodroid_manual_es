DESCARGA DE DATOS

El botón Descargar inicia una conexión con el dispositivo para descargar datos. Mientras la conexión está activa, permanece en rojo. Si está utilizando el modo de descarga bajo demanda, la conexión se cierra cuando no hay más datos para recuperar. Con el modo de descarga continuo, la conexión permanece abierta (y los datos se descargan continuamente tan pronto como se toman) hasta que el dispositivo y el Android pierdan la conexión Bluetooth. El modo que mejor se adapte a ti depende de tus hábitos. Con el modo bajo demanda, puede concentrarse en dibujar y otras tareas sin que le molesten los datos entrantes, y cuando los descarga puede concentrarse en ellos (están resaltados en azul). Por otro lado, la descarga bajo demanda requiere pausas para obtener los datos, y estos son especialmente largos si tomaste varias fotos en el medio.

Cuando se interrumpe la conexión Bluetooth con el dispositivo (p. ej., el dispositivo se aleja demasiado del Android o está apagado) en el modo bajo demanda, la descarga de datos se interrumpe. En modo continuo, TopoDroid vuelve a intentar conectarse al dispositivo cada pocos segundos.

Cuando la ventana de datos de la topografía se cierra, la conexión de descarga de datos finaliza. La conexión de descarga no se interrumpe cuando el la ventana está en pausa (la pantalla se apaga). Si abre un croquis mientras la conexión de descarga está activa en modo continuo, la conexión se mantiene y los datos se descargan (y se muestran en el croquis).

Para Sap5 y Bric4 los datos siempre se descargan en modo continuo.

Bric4
La descarga de datos del Bric4 se puede limitar a los valores esenciales (distancia, rumbo, inclinación y poco más), o completa, con toda la información transferida por el protocolo Bric4.
El protocolo completo incluye el "índice" de la toma. Por defecto, TopoDroid usa el índice Bric4 como índice de datos en la tabla de la base de datos, si lo recibe. Dado que el índice de datos debe ser único, si TopoDroid recibe una toma con un índice que ya está en la base de datos, se utiliza un valor novedoso en lugar del índice Bric4.
Es posible deshabilitar el uso del índice Bric4 [G], y dejar que TopoDroid asigne valores incrementales al índice de datos.

ADVERTENCIA. El primer disparo después de un "borrado de memoria" de Bric4 no llega a TopoDroid.
