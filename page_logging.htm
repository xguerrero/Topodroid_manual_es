REGISTRO de TopoDroid

El registro es importante para resolver problemas cuando el programa no se comporta como se esperaba. Por ejemplo, cuando el resultado de un cálculo no concuerda con el obtenido por otros medios, o una entrada del usuario no se analiza correctamente.

El registro se controla desde el menú "Registro" de la ventana principal.
Los mensajes de registro se pueden enviar a system log-stream, o a un archivo de registro, que se guarda en la carpeta base de TopoDroid. El registro del sistema se puede insertar desde una PC a través de USB con programas como "adb". El archivo de registro se puede abrir en "modo de adición", con mensajes de registro adjuntos a los registros existentes, o" modo de sobrescritura".

Hay varios indices de registro para limitar los registros a los necesarios para encontrar la causa del problema. De forma predeterminada, solo se registran los "errores" (es decir, cuando el programa detecta algo que no debería haber ocurrido).

Los registros [E] son ​​diferentes de los bloqueos de software. Los bloqueos se deben a que el programa se comporta mal en el sistema y son detectados por el sistema, que finaliza el programa. Debe informar los bloqueos (y ANR) a través de Google Play para que se adjunte un seguimiento de la pila y se puedan solucionar más fácilmente.
Consulte el sitio web de TopoDroid para obtener informes sobre bloqueos y ANR.
